## 1. 背景与目标

高中数学《指数函数的图像和性质》（人教版必修一 4.2 节）现有一份完整授课逐字稿。
目标：制作一个**课堂投影讲解用的网页动态课件**，覆盖讲课稿全部 7 个教学环节，
以连续演变的数学画板替代"PPT + 几何画板"组合，老师用翻页笔/键盘控制节奏。

**成功标准**：
- 教室电脑双击 `index.html` 即可使用，完全离线，无需安装任何环境；
- 图像在同一坐标系上跨环节连续演变（描点 → 连线 → 翻折 → 一族曲线 → 性质标注）；
- 任意顺序前进/后退/跳转，画面状态始终正确；
- 投影大屏（含 1024×768 老设备）上后排学生看得清。

## 2. 非目标（明确不做）

- 学生答题与判分功能；
- 移动端/触屏专门优化（定位是教师投影）；
- 录课自动播放模式；
- 多课程课件平台框架（仅聚焦本节课）。

## 3. 技术选型

| 项 | 选择 | 理由 |
|----|------|------|
| 基础栈 | 原生 HTML/CSS/JS，无构建 | 零环境依赖，拷贝即用 |
| 数学画板 | JSXGraph（本地化 vendor/） | 教学级坐标系/曲线/滑块/动点，开源免费 |
| 公式排版 | KaTeX + 本地字体（vendor/） | 轻量快速，离线可用 |
| 动画 | 自写 tween.js（rAF + 缓动，约50行） | 避免引入 GSAP，减少依赖 |
| 幻灯框架 | 不用（自建步进导演） | 画板需跨环节连续，幻灯片"一页一景"模型不适配 |
| 目标浏览器 | Chrome / Edge 现代版 | 教室主流；不支持 IE |

## 4. 架构

### 4.1 文件结构

```
math-l/
├── index.html               入口（file:// 双击直开）
├── vendor/
│   ├── jsxgraph/            jsxgraphcore.js + jsxgraph.css
│   └── katex/               katex.min.js/css + fonts/
├── css/main.css             布局、主题、投影可读性样式
└── js/

    ├── director.js          导演：光标(环节,步)、键盘映射、进度UI、hash同步
    ├── stage.js             舞台：JSXGraph 封装 + 语义化动画 API
    ├── panel.js             面板：讲解文字/KaTeX/表格/控件渲染
    ├── tween.js             rAF 缓动工具
    └── scenes/

        └── s7-population.js 人口增长应用与小结
```

### 4.2 导演—剧本—舞台—面板模型

- **剧本（scene）**：`{ id, title, setup(stage, panel), steps: [step…] }`；
  **步（step）**：`{ narration: '面板讲解文案（含 $..$ 公式）', enter(animate) }`。
  `enter(true)` 带动画推进到本步终态；`enter(false)` 直接落到终态（供快放）。
- **导演（director）**：维护 `(sceneIndex, stepIndex)`；前进 = 当前步 `enter(true)`；
  后退/跳转 = 重置画板 → 目标环节 `setup()` → 快放 `enter(false)` 至目标步。
  位置实时写入 URL hash（如 `#4/2`），刷新后自动恢复。
- **舞台（stage）**：管理 board 生命周期与对象注册表；语义化 API：
  `plotCurve(fn, opts)` 曲线生长 · `dropPoint(x,y)` 落点 · `mirrorFold(id)` 沿Y轴翻折
  · `slider(cfg)` 底数滑块 · `movingPoint(curveId)` 沿线动点 · `flyTo(bbox)` 视窗飞行
  · `highlight(id)` / `shade(region)` 高亮与区域着色 · `reset()` 清场。
- **面板（panel）**：`setTitle() / setNarration() / renderTable() / renderCard()`；
  底部控制条含"上一步/下一步"按钮与快捷键提示。

## 5. 关键技术决策

| # | 决策 | 理由与代价 |
|---|------|-----------|
| D1 | 不用 ES Modules，普通 `<script>` 顺序加载 + `window.CW` 命名空间 | file:// 下 ESM 被 CORS 拦截会导致双击打开失败；牺牲模块化优雅换零门槛 |
| D2 | 后退/跳转 = 重置 + 无动画快放 | 每步只写 enter()，无需逆操作；任意操作序列状态一致 |
| D3 | URL hash 记忆进度 | 课堂误刷新 1 秒恢复原位 |
| D4 | 不引入 GSAP | 需求仅缓动/序列，自写 50 行即可，少一个故障点 |
| D5 | 语义配色恒定：a>1 暖色（红橙）、0<a<1 冷色（蓝青） | 全课件视觉锚点；红蓝对比对色觉障碍友好 |
| D6 | 键位对齐翻页笔：PgUp/PgDn 即上一步/下一步 | 翻页笔无需任何适配 |

## 6. 教学分镜明细（7 环节 · 约 32 步）

> 每步对应老师按一次翻页笔；"面板"列为讲解要点（实现时配完整话术与公式）。

### S1 复习导入（3 步）
| 步 | 画板动画 | 面板 |
|----|---------|------|
| 1 | 空坐标系淡入 | 课题 + 定义卡片 $y=a^x\ (a>0,a\neq1)$ |
| 2 | — | 高亮：x 为自变量，定义域为 R |
| 3 | — | 研究路径：类比幂函数（定义→图像→性质），先研究 y=2ˣ 与 y=(1/2)ˣ |

### S2 描点法作图（6 步）
| 步 | 画板动画 | 面板 |
|----|---------|------|
| 1 | — | 空数值表出现（x=-2,-1,0,1,2 两函数行） |
| 2 | — | y=2ˣ 行逐格弹入，演示 2⁻²=1/4 计算 |
| 3 | — | y=(1/2)ˣ 行逐格弹入 |
| 4 | 数值逐个从表格"飞落"为坐标系中的点 | 对应表格列同步高亮（步骤：描点） |
| 5 | 红色 y=2ˣ 曲线生长；再蓝色 y=(1/2)ˣ 生长 | 步骤：连线（平滑曲线顺次连接） |
| 6 | 两曲线完成态 + 函数标签 | 引导观察两图像关系 |

### S3 对称性（3 步）
| 步 | 画板动画 | 面板 |
|----|---------|------|
| 1 | 对应点对高亮（如 (1,2)↔(-1,2)），水平虚线连接 | 发现：点关于 Y 轴对称 |
| 2 | y=2ˣ 半透明副本沿 Y 轴翻折，与 y=(1/2)ˣ 重合闪烁 | 两图像关于 Y 轴对称 |
| 3 | — | 结论卡片：底数互为倒数 ⇒ 图像关于 Y 轴对称（一般结论） |

### S4 底数滑块探索（5 步）★ 替代几何画板的核心环节
| 步 | 画板动画 | 面板 |
|----|---------|------|
| 1 | 一族浅色背景曲线生长（a=3,4,1/3,1/4…） | 两个函数不够，需观察更多 |
| 2 | 滑块 a∈[0.1,5] 出现，主曲线随拖动实时形变；a=1 时退化为水平线 y=1 | 顺势回答"为什么 a≠1"；显示当前 a 值 |
| 3 | 自动演示 a:1.5→5，x>0 区域高亮 | Y 轴右侧：底大幅高（a>1） |
| 4 | 自动演示 a:0.9→0.15，x<0 区域高亮 | Y 轴左侧：底大图低（0<a<1） |
| 5 | 全族曲线按类着色：a>1 暖色 / 0<a<1 冷色 | 结论：分两类研究 |

### S5 性质汇总（6 步）
| 步 | 画板动画 | 面板 |
|----|---------|------|
| 1 | 沿 X 轴扫描光带；X 轴上方区域着色 | 定义域 R；值域 (0,+∞)，图像恒在 X 轴上方 |
| 2 | 滑块扫动，(0,1) 点全程脉冲高亮 | 过定点 (0,1)：$a^0=1$ |
| 3 | 动点沿 a>1 曲线左→右爬升；切换 0<a<1 曲线下滑 | 单调性：a>1 增函数；0<a<1 减函数 |
| 4 | 动点跑向远端，视窗跟随，曲线贴近 X 轴不相交 | 渐近性；无最大值最小值 |
| 5 | 快速翻折（Y 轴）与旋转 180°（原点）演示均不重合 | 非奇非偶函数 |
| 6 | — | 性质对照总表逐行点亮（0<a<1 列 vs a>1 列） |

## 7. 布局与视觉规范

```
┌────────────────────────────────────────────┐
│ 课题名                    ●●○○○○○ 环节进度   │
├───────────────────────┬────────────────────┤
│  数学画板（约62%）       │ 讲解面板（约38%）     │
│                       │  标题/公式/表格/控件  │
├───────────────────────┴────────────────────┤
│ ◀ 上一步 · 下一步 ▶ · 环节 n/7 步 m/k · 全屏 │
└────────────────────────────────────────────┘
```

- 16:9 优先，弹性适配至最低 1024×768；
- 浅色高对比主题；正文 ≥20px 随视口缩放（clamp），公式更大一号；曲线线宽 ≥3px；
- 配色遵循 D5：a>1 暖色 / 0<a<1 冷色，贯穿全部环节。

## 9. 容错设计

- **离线**：所有库与字体本地化，断网零影响；
- **状态一致性**：D2 重置+快放机制，任意操作序列不出现错乱画面；
- **误刷新恢复**：URL hash 记忆 `(环节,步)`，重载后 1 秒内恢复；
- **降级**：JSXGraph 初始化失败时显示明确错误提示（提示换用 Chrome/Edge）。

## 10. 验收清单

1. macOS/Windows 上 `file://` 双击直开，控制台零报错，断网可用；
2. 全部步骤顺序前进一遍、逐步后退一遍、数字键随机跳转 20 次，画面均正确；
3. 数学正确性：数值表、曲线形状、性质结论与教材一致；
4. 1024×768 与 1920×1080 布局不破版，后排可读；
5. 逐步截图走查全部约 32 步的视觉效果（开发期用无头浏览器）。
